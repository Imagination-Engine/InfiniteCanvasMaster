# Conductor

This directory manages the specs, plans, and progression of the capstone project via the Conductor framework.

## How to use this
Use the Conductor CLI to initialize tracks and run through the workflow. `tracks.md` stores the state of ongoing tracks.

## Structure
- `tracks/`: Contains individual track spec and plan files.
- `*.md`: Various global definitions like product, tech-stack, and workflow.

## Standards
Conductor workflow rules apply.
