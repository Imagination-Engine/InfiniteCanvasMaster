# Studio Capability Manifests Spec Pack

Generated: 2026-05-04.

This pack defines the next phase of the Imagination Canvas: each studio has a rigorous manifest, but the canvas can mix blocks, tools, agents, runtimes, artifacts, and models from any studio in a single workspace.

## Core Principle

Studios are not silos. They are capability domains.

The canvas composes capabilities across studios. A script from Writer's Studio can feed Video Studio. A world bible can feed Game Studio. A checkout flow can feed App Creation Studio. Research can feed everything. Agent/Automation Studio can create helpers for any other studio.

## How To Use

Give the implementation agent:

```txt
/docs/studio-capability-manifests/21-master-implementation-prompt.md
```

The agent must first create a current capability matrix, then generate granular tracks/tasks/subtasks, then implement one vertical studio slice at a time.
