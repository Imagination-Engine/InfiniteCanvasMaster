# Imagination Canvas Extraction + Implementation Pack

This document set translates the best interaction patterns from tldraw-style infinite canvas systems and AFFiNE-style edgeless/page hybrid workspaces into a Balnce-native Imagination Canvas specification.

The goal is not to copy either product. The goal is to extract battle-tested interaction grammar and rebuild it as a sovereign, agentic creation surface where every block can become a note, artifact, app, workflow, agent, goal, memory cluster, or commerce object.

## How to use this package

1. Start with `01-canvas-philosophy-and-target-state.md`.
2. Use `04-object-model-and-content-grammar.md` and `17-balnce-native-blocks.md` as the canonical domain model.
3. Use `16-component-implementation-plan.md` as the engineering plan.
4. Use `18-master-extraction-and-implementation-prompt.md` as the full agent/coding-team execution prompt.
5. Use `15-gap-list.md` as the running production-readiness checklist.

## Reference targets

- tldraw: infinite canvas interaction engine, shapes, camera, selection, bindings, tools, multiplayer-friendly architecture.
- AFFiNE: edgeless/page hybrid knowledge surface, block-based content, structured documents living inside infinite spatial workspaces.
- Balnce: personal AI OS, Imagination Engine, artifact blocks, agent blocks, local models, Edge Twin, goals, memory, identity, AURA, intentcasting, provenance, and sovereign creation.

## Non-negotiable product bar

The Imagination Canvas must feel calm, spatial, precise, cinematic, and alive. It must never feel like a generic whiteboard, a static dashboard, or boxes floating on a plane.
