# DAG to Block Layout Plan

## Objective

Enable the canvas to visually emerge from a compiled plan/DAG generated by the orchestrator, rather than requiring manual placement for everything.

## Components

1. **useDagToBlocks**: A hook/utility that takes a JSON plan (e.g., from a Mastra tool call) and instantiates the required typed blocks.
2. **CanvasLayoutCompiler**: An algorithm that arranges the instantiated blocks.

## Layout Algorithms

- **Tree**: Top-down or left-to-right. Good for hierarchical plans.
- **Pipeline**: Strict left-to-right flow. Good for sequential agent tasks.
- **Swarm Cluster**: A central orchestrator block surrounded by specialized agents.
- **Studio Board**: A grid or masonry layout grouped by studio category.

## Visuals

- Use Framer Motion to stagger the appearance of the blocks.
- Draw pulsing, glowing intention/dependency lines between them to show the data or control flow.
