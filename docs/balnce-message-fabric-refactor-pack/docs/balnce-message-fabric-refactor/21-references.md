# 21 — References

These references ground transport and collaboration assumptions.

- MDN EventSource / SSE: SSE is unidirectional server-to-client and suited to cases where the client does not need to send messages back on the same channel. https://developer.mozilla.org/en-US/docs/Web/API/EventSource
- MDN Server-Sent Events: server can push messages to a web page at any time. https://developer.mozilla.org/en-US/docs/Web/API/Server-sent_events
- Yjs docs: Yjs is a CRDT/shared data type framework for collaborative applications; Awareness handles user status, cursors, and presence. https://docs.yjs.dev/
- Yjs Awareness API: https://docs.yjs.dev/api/about-awareness
- tldraw collaboration docs: tldraw SDK supports real-time collaboration with `@tldraw/sync`. https://tldraw.dev/docs/collaboration
- tldraw sync docs: sync provides fast, fault-tolerant shared document syncing; production use requires self-hosting. https://tldraw.dev/docs/sync
- gRPC official docs: high-performance RPC framework with load balancing, tracing, health checking, and authentication support. https://grpc.io/
- ZeroMQ official docs: embeddable messaging/concurrency framework with in-process, inter-process, TCP, multicast, pub-sub, fan-out, task distribution, and request-reply patterns. https://zeromq.org/
- MDN WebTransport API: HTTP/3-based API supporting reliable and unreliable transports. https://developer.mozilla.org/en-US/docs/Web/API/WebTransport_API
